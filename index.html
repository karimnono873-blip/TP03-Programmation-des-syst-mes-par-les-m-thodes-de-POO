Analyse Structurelle : De la RAM au Processeur

Ce TP n'est pas qu'un simple exercice de syntaxe. C'est une application concrète des concepts fondamentaux de l'ingénierie logicielle. L'objectif est de programmer un système contraint, en gérant manuellement les allocations mémoire et en prévenant les erreurs fatales d'exécution.

1. Allocation Dynamique (Java) vs Allocation Statique (C++)

Comme le rappelle l'énoncé du TP, en C++, un tableau classique est statique : int tab[10];. Sa taille doit être connue par le compilateur avant l'exécution. C'est rigide et cela peut gaspiller de la mémoire. En Java, l'approche est orientée objet et dynamique.

L'instruction plateau = new char[taille]; est le cœur de notre optimisation. À la compilation, la variable plateau n'est qu'une simple référence (un pointeur) stockée dans la Pile (Stack), qui ne pèse presque rien. Ce n'est qu'à l'exécution (Runtime), une fois que l'utilisateur a entré son nombre de pions, que le mot-clé new s'active. Il demande au système d'exploitation d'allouer l'espace exact nécessaire dans le Tas (Heap). Le programme est ainsi parfaitement optimisé pour sa consommation RAM.

2. Prévention des Failles et Complexité Algorithmique

La cybersécurité et la stabilité d'un logiciel commencent par la validation des entrées. La ligne if (pos >= 0 && pos < plateau.length) n'est pas optionnelle. Elle agit comme un pare-feu physique. Si on laisse un utilisateur accéder à l'indice -1 ou 15 sur un tableau de 10, le processeur va tenter de lire une adresse mémoire non allouée. En C, cela causerait un "Segmentation Fault". En Java, cela déclenche la fameuse ArrayIndexOutOfBoundsException qui fait crasher l'application. Notre condition l'empêche.

De plus, nos fonctions de vérification d'état (estGagne(), estBloque()) scannent le tableau avec des boucles for. Leur complexité algorithmique temporelle est de O(N). Pour optimiser le temps de calcul du processeur, j'ai utilisé des évaluations en court-circuit. Dès qu'un pion n'est pas à sa place, la fonction exécute return false;. La boucle est immédiatement détruite, économisant des millions de cycles d'horloge sur de grandes données.

3. Le Cycle de Vie des Objets et le Garbage Collector

L'outil Scanner sc = new Scanner(System.in); crée un objet lourd qui s'interface avec le flux matériel du système d'exploitation (le clavier). Maintenir cette connexion consomme de l'énergie et de la mémoire. C'est pourquoi la dernière instruction de notre programme est sc.close();.

Cette instruction coupe la liaison matérielle. Notre objet Scanner devient alors une "île isolée" dans la mémoire. Le gestionnaire de mémoire de Java, appelé le Garbage Collector (Ramasse-miettes), va scanner la RAM en arrière-plan, détecter cet objet devenu inutile, le détruire, et rendre la mémoire vive au système d'exploitation. C'est la garantie d'un code sans aucune fuite de mémoire (Zero Memory Leak).

Dissection Instruction par Instruction

Voici l'analyse granulaire du code, justifiant chaque choix architectural et expliquant son impact direct sur l'expérience de l'utilisateur.

Segment de Code (Java) Mécanique Interne & Raisonnement (Ingénierie)
final int NbMAX = 10; Sécurité des constantes : Le mot-clé final verrouille l'espace mémoire. La variable devient inaltérable, protégeant l'algorithme contre des modifications accidentelles ou des dépassements de tampon (Buffer Overflow). Pour l'utilisateur, c'est une garantie que le jeu restera jouable et que la console ne sera pas inondée par un tableau d'un million de cases.
while (!finDeJeu) { ... } Boucle d'état du Thread principal : Cette boucle infinie conditionnelle maintient le processus éveillé. Elle ordonne au processeur de répéter le triptyque : Affichage -> Lecture du Coup -> Vérification. Elle ne se brise que lorsque la machine à état bascule le booléen finDeJeu à true.
int pos = sc.nextInt(); Suspension du Processus (Blocking I/O) : Cette ligne est vitale. Elle ordonne au système de suspendre l'exécution du programme. Le processeur ne consomme plus de ressources et attend qu'une interruption matérielle (la touche Entrée) réveille le processus. C'est le moment de réflexion pour l'utilisateur.
if (pos + 1 == indVide) Logique de glissement (Vecteur Physique) : L'Unité Arithmétique du processeur calcule l'adresse mémoire suivante (pos + 1). Si cette adresse correspond exactement à l'adresse de la case vide stockée dans indVide, le mouvement "physique" de glissement vers la droite est autorisé par les lois de notre système.
plateau[posVide] = plateau[pos];
plateau[pos] = ' ';		 Manipulation Directe de Mémoire (O(1)) : Ici réside l'illusion du mouvement. Il n'y a pas d'animation compliquée, juste un échange (Swapping) de caractères dans la RAM. On copie l'octet représentant le pion vers la nouvelle adresse mémoire, puis on écrase l'ancienne adresse avec un espace vide. Cette opération est instantanée.
if (plateau[i] != 'R')
return false;		 Optimisation de la vérification (Court-circuit) : Dans la méthode estGagne(), on vérifie si la gauche est pleine de 'R'. Si le processeur tombe sur un 'N' dès la première case, il exécute le return false. La fonction s'arrête instantanément. Il est mathématiquement inutile de vérifier le reste du tableau. C'est de l'ingénierie optimisée.

Code Source Intégral (Solution Optimisée)

import java.util.Scanner;

public class SauteMouton {

    // Variables globales stockées dans le segment de données
    static final int NbMAX = 10; 
    static char[] plateau;
    static int nbPions;
    static int indVide;
    static Scanner sc = new Scanner(System.in);

    public static void main(String[] args) {
        boolean rejouer = true;

        // Boucle système (Main Thread)
        while (rejouer) {
            initialiserPlateau();
            boolean finDeJeu = false;

            // Moteur du jeu (Game Loop)
            while (!finDeJeu) {
                afficherPlateau();
                jouerUnCoup();

                // Machine à états : vérification continue
                if (estGagne()) {
                    afficherPlateau();
                    System.out.println("\n[+] VICTOIRE ! Configuration inversée.");
                    finDeJeu = true;
                } else if (estBloque()) {
                    afficherPlateau();
                    System.out.println("\n[-] BLOCAGE ! Plus aucun mouvement possible.");
                    finDeJeu = true;
                }
            }

            System.out.print("\nRejouer ? (O/N) : ");
            String rep = sc.next();
            rejouer = rep.equalsIgnoreCase("O");
        }
        
        System.out.println("Fermeture du système.");
        sc.close(); // Libération des ressources (Garbage Collection)
    }

    static void initialiserPlateau() {
        nbPions = 0;
        while (nbPions < 1 || nbPions > NbMAX) {
            System.out.print("Nombre de pions par couleur (Max " + NbMAX + ") : ");
            nbPions = sc.nextInt();
        }

        // Allocation dynamique dans le Heap
        int taille = 2 * nbPions + 1;
        plateau = new char[taille];
        indVide = nbPions;

        // Remplissage de la mémoire (Initialisation RAM)
        for (int i = 0; i < nbPions; i++) plateau[i] = 'N';
        plateau[indVide] = ' ';
        for (int i = nbPions + 1; i < taille; i++) plateau[i] = 'R';
    }

    static void afficherPlateau() {
        System.out.print("\nIndices : ");
        for (int i = 0; i < plateau.length; i++) System.out.printf("[%2d] ", i);
        
        System.out.print("\nPlateau : ");
        for (int i = 0; i < plateau.length; i++) System.out.printf("  %c  ", plateau[i]);
        System.out.println("\n-----------------------------------------------------");
    }

    static void jouerUnCoup() {
        boolean coupValide = false;
        while (!coupValide) {
            System.out.print("Indice du pion à déplacer : ");
            int pos = sc.nextInt();

            // Sécurité : Prévention du ArrayIndexOutOfBoundsException
            if (pos >= 0 && pos < plateau.length) {
                if (plateau[pos] == 'N') {
                    // Les Noirs vont à Droite (+1 ou +2)
                    if (pos + 1 == indVide) {
                        effectuerDeplacement(pos, indVide); coupValide = true;
                    } else if (pos + 2 == indVide && plateau[pos + 1] == 'R') {
                        effectuerDeplacement(pos, indVide); coupValide = true;
                    } else System.out.println("[!] Mouvement interdit.");
                } else if (plateau[pos] == 'R') {
                    // Les Rouges vont à Gauche (-1 ou -2)
                    if (pos - 1 == indVide) {
                        effectuerDeplacement(pos, indVide); coupValide = true;
                    } else if (pos - 2 == indVide && plateau[pos - 1] == 'N') {
                        effectuerDeplacement(pos, indVide); coupValide = true;
                    } else System.out.println("[!] Mouvement interdit.");
                } else System.out.println("[!] Case vide sélectionnée.");
            } else System.out.println("[!] Erreur mémoire : Indice hors limites.");
        }
    }

    static void effectuerDeplacement(int posPion, int posVide) {
        // Swapping en O(1)
        plateau[posVide] = plateau[posPion];
        plateau[posPion] = ' ';
        indVide = posPion; // Mise à jour de la constante spatiale
    }

    static boolean estGagne() {
        // Optimisation en court-circuit (Returns immédiats)
        for (int i = 0; i < nbPions; i++) if (plateau[i] != 'R') return false;
        if (plateau[nbPions] != ' ') return false;
        for (int i = nbPions + 1; i < plateau.length; i++) if (plateau[i] != 'N') return false;
        return true;
    }

    static boolean estBloque() {
        // Recherche exhaustive d'au moins un mouvement possible
        for (int i = 0; i < plateau.length; i++) {
            if (plateau[i] == 'N') {
                if (i + 1 == indVide || (i + 2 == indVide && plateau[i + 1] == 'R')) return false;
            } else if (plateau[i] == 'R') {
                if (i - 1 == indVide || (i - 2 == indVide && plateau[i - 1] == 'N')) return false;
            }
        }
        return true; 
    }
}
